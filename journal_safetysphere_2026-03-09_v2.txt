3. MÉTHODOLOGIE DE DÉVELOPPEMENT (nouvelles règles v2.0.0)
────────────────────────────────────────────────────────────────────────────────

Depuis la refacto, chaque session suit ce protocole :

  1. Fournir à Claude : CONTEXT.md + journal + fichiers JS concernés (1 à 3 max)
  2. Claude génère uniquement les fichiers modifiés
  3. Remplacement sur GitHub via crayon ✏️ (Edit file)
  4. Vercel déploie automatiquement en 30 secondes

Règle : on ne fournit JAMAIS tous les fichiers — uniquement ceux touchés par la feature.

  Type de tâche                    → Fichiers à fournir
  Nouvelle fonctionnalité isolée   → Aucun ou 1 fichier
  Modifier un module existant      → Le fichier du module
  Bug multi-modules                → Les 2-3 fichiers concernés
